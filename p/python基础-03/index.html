Pandas 简介

Pandas 是由量化专家 Wes McKinney 开发的 Python 数据分析库，其名称源于“面板数据”与“数据分析”的缩写组合。它在经济金融领域尤为重要，被视为处理以日期或时间为索引的时间序列数据的最佳利器。除了强大的时间序列功能外，Pandas 还提供了高性能、易于使用的数据结构（如 Series 和 DataFrame），能够极大地简化数据清洗、准备、处理和分析的复杂流程。其主要用途非常广泛，涵盖了从读取多种格式文件（如 CSV、Excel、SQL）开始，进行缺失值填补、数据筛选与对齐、合并重塑，到执行复杂的统计分析、分组聚合运算，最终辅助用户完成高质量的数据可视化，是进行数据挖掘、量化交易策略研究及各类商业智能分析不可或缺的基础工具。

Pandas 文件导入

在开始之前先简单区分一下，CSV本质上是一个纯文本文件，虽然它可以用Excel打开，但并不像XLSX那样包含复杂的格式、公式等额外信息；正因为CSV只单纯存储数据，所以在读取速度上通常比XLSX更快。接下来，我们就以这种常见的 CSV 文件格式为例，为大家演示如何使用 Pandas 读取数据并进行基础的数据分析。

df = pd.read_csv('002415.csv',   # 这里输入文件的位置参数，一般会把数据文件放在工作文件夹下
                  encoding = 'gbk',   # encoding参数可以设定编码方式，一些中文数据文件有时需要使用'gbk'编码
                  n_rows = 10  # 设定为读取前10行，默认为读取全部数据
                  index_col = ['date'],   # 这里设定DataFrame的索引列，如果不设定pandas会自动生成0，1，2...作为DataFrame的索引
                  usecols = ['date','open','high','close','low','volume'], # 这里选定要读取的列
                  na_values = 'NULL' )  # 这里是将'NULL'识别为缺失值，如果不设置read_csv会将'NULL'作为字符串读入
df.head()  # 显示数据前几行，方便检查是否正确读入数据，默认显示前5行

以上是读取数据时一些常见的参数设置。通常读取完文件后，我们会得到一个 DataFrame 对象。但在处理超大数据量的文件时，可能无法一次性将其全部加载到内存中，这时就需要用到 `chunksize` 参数。一旦设置了该参数，`read_csv` 返回的就不再是普通的 DataFrame，而是一个迭代器。这个特性在分布式计算或大规模数据处理中非常实用，不过鉴于我们目前主要讲解 Python 基础，这里就不做深入展开了，感兴趣的同学可以进一步查阅分布式计算的相关资料。 除了 CSV 文件之外，其实 Pandas 还支持读取多种类型的数据。包括我们刚才提到的 Excel，以及 SAS 等统计软件常用的数据格式；此外，像 Stata 的 .dta 文件，甚至 MySQL 等数据库中的数据，Pandas 也都能轻松读取。可以说，Pandas 在数据导入方面的兼容性和覆盖面是非常广泛的。

DataFrame和Series基础

Series

Series 非常类似于 Excel 表格中的单独一列，它能够存储大量的数据，且这些数据通常具有相同的数据类型。这一列数据并非孤立存在，而是与一个索引紧密绑定，我们在读取或创建数据时往往会直接设定好这个 Index，从而建立起数据与标签之间一一对应的关系。比如在我们刚才的例子中’open’就是一个Series.

这种结构最大的优势在于它赋予了数据“自我描述”的能力。相比于 Python 原生的列表，Series 不仅仅是一堆数值的集合，它通过预设的索引将每个数值锚定在特定的位置或含义上。这使得我们在后续的数据分析中，可以直接通过索引标签来精准定位数据，而无需关心它在内存中的物理位置，极大地提升了代码的可读性和数据处理的效率。

DataFrame

核心定义

DataFrame 是一个二维的表格型数据结构，你可以把它直接想象成一张完整的 Excel 表格或者数据库里的一张数据表。如果说 Series 是单列的一维数组，那么 DataFrame 就是由多个 Series（也就是多列）横向拼在一起组成的二维容器，它是 Pandas 中最常用、最核心的数据处理工具。

三大组成部分

一个标准的 DataFrame 包含三个关键要素：

  • 行索引 (Index)：和 Series 一样，每一行都有一个对应的标签（比如日期、ID等），用于唯一标识这条记录。
  • 列名 (Columns)：这是 DataFrame 独有的。每一列都有自己的名字（比如“姓名”、“年龄”、“销售额”），这些列名其实就是内部包含的各个 Series 的名字。
  • 数据 (Data/Values)：表格中实际存储的具体数值矩阵。

核心特征

  • 异构数据类型：在同一个 DataFrame 里，不同的列可以存储不同类型的数据。比如，“姓名”列可以是字符串，“年龄”列是整数，“是否在职”列是布尔值，它们能完美地共存于同一张表中。
  • 自动对齐：当你对两个 DataFrame 进行运算时，Pandas 会自动根据行索引和列名去匹配数据，完全不需要你手动去对齐位置。
  • 强大的操作能力：它内置了极其丰富的功能，比如筛选特定的行或列、按某列排序、分组统计、处理缺失值以及合并多张表格等。

总结

简单来说，Series 是构成 DataFrame 的基本单元，而 DataFrame 就是多个共享同一套行索引的 Series 组成的字典。在日常的数据分析工作中，我们绝大部分时间都是在和 DataFrame 打交道，因为它最符合我们看报表和处理业务数据的直觉。

pd.merge() 函数具备自动检测两个 DataFrame 公共列名的能力，并默认将其作为连接键（Key）执行合并操作，这在逻辑上等同于 SQL 中的内连接。此外，该函数的 left_on 与 right_on 参数设计与 SQL 的 JOIN ... ON 语法高度一致，允许用户在列名不一致的情况下，显式指定“左表列等于右表列”的匹配条件，从而实现精准的表关联。
